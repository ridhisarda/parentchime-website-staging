# Parentchime Website Prototype

## Current Prototype Structure

1. Hero
   - Main promise: a child-centered tracking system for nutrition, supplements, health, growth, school, and co-parenting.
   - Early parent waitlist call-to-action.
   - Real product screenshots from launch assets.

2. Why Tracking Matters
   - Credible proof points from HHS / APA, Springer, India TV, and UNICEF.
   - Frames the issue as parent stress, micronutrient gaps, diabetes risk, and rising childhood obesity.

3. Why Adult Apps Do Not Solve This
   - Adult fitness apps are built around personal weight goals.
   - Parentchime is built around changing pediatric needs and parent workflows.

4. Founder Story
   - Placeholder copy to be replaced with the founder story verbiage.
   - This should become one of the most emotionally persuasive sections.

5. Features
   - Nutrition versus child-specific targets.
   - Supplement and multivitamin tracking.
   - Health records.
   - School planner.

6. Pro Positioning
   - 5 PM nutrition check.
   - Food ideas for nutrition gaps.
   - Shared family view.
   - Community help and feedback.
   - Custom recipe saver.
   - Image capture for easy nutrition tracking.
   - Micronutrient insights.

7. Contact and Sources
   - Contact email placeholders.
   - Source links in footer.

## Required Inputs Before Launch

- Final founder story copy.
- Final support/contact email address.
- Brand domain choice, e.g. `parentchime.com`.
- Decision on whether homepage should focus India-first or global parent audience.

## Deployment Plan

### Option A: Vercel

Best if we later add a blog, lead capture form, analytics, or React components.

1. Create a small web app project from this static prototype.
2. Push to GitHub.
3. Import project in Vercel.
4. Add custom domain.
5. Add analytics and SEO metadata.
6. Replace app links after stores approve.

### Option B: Netlify

Best for a simple static marketing site with easy forms.

1. Deploy the `website` folder as a static site.
2. Configure domain and SSL.
3. Use Netlify Forms for early access / contact collection.
4. Replace app links after stores approve.

### Option C: Supabase Storage / Static Hosting

Good enough for a temporary pre-launch page, but less flexible for marketing.

Recommended path: start with Vercel or Netlify.
